# デジハク生成AI LP 忠実再現 設計書

## 目的

公開中の `https://ai.digital-hacks.jp/` を参照し、Next.jsで視覚・レイアウト・レスポンシブ表示・主要インタラクションを忠実に再現する。完成後は元LPとセクション単位で比較し、類似度95%以上を合格基準とする。

## 技術構成

- Next.js App Router
- TypeScript
- React
- CSS ModulesとグローバルCSS変数
- Vitest + React Testing Library
- PlaywrightによるPC・タブレット・スマートフォン表示検証
- 必要な場合のみ `image_gen` によるビジュアル素材生成

## 再現対象

元LPの主要構成を、次の独立したセクションとして実装する。

1. ヘッダー、ロゴ、グローバルナビゲーション
2. ファーストビュー、メインコピー、人物ビジュアル、実績数値
3. キャンペーン・返金保証表示
4. 課題提起
5. 「デジハクとは」
6. 口コミ・実績表示
7. 特徴セクション
8. 受講者事例
9. カリキュラム・サポート紹介
10. 料金表
11. 受講までの流れ
12. FAQ
13. 最終CTAとフッター

## コンポーネント設計

- `app/page.tsx`: セクションの順序とページ全体の組み立てのみを担当する。
- `components/layout`: ヘッダー、フッター、共通コンテナを担当する。
- `components/sections`: 各LPセクションを1ファイル単位で分離する。
- `components/ui`: CTA、見出し、数値表示、カード、FAQ項目など再利用可能な表示部品を担当する。
- `content/lp-content.ts`: 文言、数値、料金、FAQ、事例データを一元管理する。
- `public/assets`: 画像、動画、フォント、生成素材をローカル管理する。

各セクションは、他セクションの内部実装に依存せず、データと表示を分離する。これにより、類似度が95%未満だったセクションだけを安全に修正できる構造にする。

## 素材方針

1. 元LPで公開配信されている画像・フォント・動画素材は、取得可能な範囲でローカルへ保存する。
2. 広告、解析、行動追跡、外部フォーム、Framer実行スクリプトは複製しない。
3. CSSやHTMLだけでは忠実な再現が難しく、元素材を取得できない装飾・合成ビジュアルに限り、`image_gen` で近似素材を生成する。
4. 生成素材には文字を焼き込まず、可能な限りテキストをHTMLとして保持する。装飾と写真の一体化が必要な場合だけ画像として扱う。
5. 実行時は外部画像へホットリンクせず、ローカル素材のみで描画する。

## レスポンシブ設計

元LPの表示を基準に、少なくとも以下の幅で忠実性を検証する。

- デスクトップ: 1280px
- タブレット: 768px
- スマートフォン: 390px

デスクトップは最大幅約1000〜1232pxのセクションコンテナを使い、元LPの余白と縦方向のリズムを再現する。スマートフォンでは見出し、人物画像、カード、料金表、CTAを縦積みに切り替え、元LPと同じ表示順を維持する。

## インタラクション

- ナビゲーションはページ内アンカーへスムーズスクロールする。
- FAQはクリックまたはキーボード操作で開閉できる。
- 口コミ、講師、実績などの自動横スクロール表示は元LPの速度と方向に合わせる。
- CTAは元LPと同じ見た目を維持するが、外部遷移、フォーム送信、LINE起動を一切行わない。
- CTAは `aria-disabled="true"` を付与し、クリックとEnter/Space操作を無効化する。
- `prefers-reduced-motion` が有効な場合は、自動スクロールと装飾アニメーションを停止する。

## エラーハンドリング

- 画像読み込みに失敗した場合は、レイアウト寸法を維持した代替背景を表示する。
- 動画が再生できない場合は同一比率の静止画を表示する。
- FAQや配列データが空でもページ全体が例外停止しないよう、セクション単位で空状態を扱う。
- 外部通信に依存しないため、第三者サービス停止時もページ表示を維持する。

## TDD方針

実装コードより先に、次の振る舞いをテストする。

- 全主要セクションが正しい順序で表示される。
- ナビゲーションが対応するセクションIDを参照する。
- CTAが外部遷移・送信を行わない。
- FAQが開閉でき、ARIA状態が同期する。
- コンテンツデータが欠けてもページが例外停止しない。
- アニメーション軽減設定で自動アニメーションが停止する。

ユニットテスト合格後、Playwrightで1280px、768px、390pxのスクリーンショットを取得する。

## 類似度検証と修正ループ

1. 元LPと実装LPを同一ビューポート・同一スクロール位置でセクション別に撮影する。
2. 実装完了後、ユーザーの指定どおりサブエージェントへ比較検証を依頼する。
3. サブエージェントは各セクションを100点満点で評価し、レイアウト、色、タイポグラフィ、素材、余白、装飾、レスポンシブ挙動の差分を報告する。
4. 95点未満のセクションは、報告された差分を根拠にメインエージェントが修正する。
5. 修正後に再度同条件で撮影し、対象セクションが95点以上になるまで検証と修正を繰り返す。
6. 全主要セクションが95点以上、ユニットテスト・ビルド・ブラウザ確認が成功した時点で完成とする。

評価の再現性を高めるため、比較時のビューポート、スクロール位置、フォント読み込み完了、アニメーション停止状態を固定する。主観評価だけでなく、ピクセル差分と構造上の差異も併用する。

## セキュリティと禁止事項

- `.env`、秘密鍵、認証情報へアクセスしない。
- APIキー、パスワード、トークンをコードへ含めない。
- 解析タグ、広告タグ、トラッキングピクセルを実装しない。
- 元LPの外部フォームや予約ページへ情報を送信しない。
- CTAから外部サイトへ遷移しない。
- `git push` は実行しない。

## 完成条件

- Next.jsでローカル起動できる。
- 元LPの主要セクション、文章、画像、レイアウト、配色、余白、主要動作が再現されている。
- 1280px、768px、390pxで表示崩れがない。
- CTAが完全に無効化されている。
- 全自動テストと本番ビルドが成功する。
- サブエージェント検証で全主要セクションが95%以上である。
- セキュリティ禁止事項に抵触していない。
