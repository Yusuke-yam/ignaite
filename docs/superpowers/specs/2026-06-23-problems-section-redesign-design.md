# セクション2「お悩み」再設計

## 目的

セクション2をAI学習上の課題提示から、人生・キャリアの停滞感を言語化する6つの悩みへ変更する。人物を視覚的な中心に置き、6つの吹き出しが人物へ向かう構図で共感を促す。

## 見出し

- 表示文言は「こんなお悩みありませんか？」とする。
- 旧文言「AIに興味はあるけど、ChatGPT触っただけで終わってませんか？」は削除する。
- 水平方向はセクション中央に配置する。
- 現在の見出し位置を基準に、セクション上端側へ約30%移動した印象になるよう、セクション上余白と見出し下余白を調整する。

## 悩み文言

次の6件をこの順序で表示する。

1. 毎日同じ日々が繰り返され人生がつまらない
2. 自分が今後何をしたいのか分からない
3. 生き方の正解や成功方法を、他人・SNS・本に探してしまう
4. 「いつか何かできたらいいな」と思いながら、何も変わっていない
5. 自己啓発本は好きでよく読むが、現実は何も変わっていない
6. 副業や個人事業に憧れるが、「どうせ自分には無理」と思い込んでいる

旧4件の悩み文言はすべて削除する。

## Desktopレイアウト

- CSS Gridで「左吹き出し3件｜人物画像｜右吹き出し3件」の3列を構成する。
- 人物画像は上下位置を現行デザインから大きく変えず、水平方向のみセクション中央へ移動する。
- 左列の吹き出しは右向きの先端、右列の吹き出しは左向きの先端を持ち、すべて人物方向を向く。
- 吹き出しは白背景、既存の淡い赤系ボーダーを継承し、角丸とCSS疑似要素による三角形の先端を持つ。
- 吹き出し内の文字量が異なっても、人物を中心とした左右の視覚バランスを維持する。
- 人物画像は既存素材 `ASSETS.problemIllustration` を再利用し、新規画像生成は行わない。

## Mobileレイアウト

- 809.98px以下では「見出し → 人物画像 → 吹き出し6件」の1列へ切り替える。
- 人物画像は中央揃えとする。
- 吹き出しは本文幅いっぱいに近いカードとして縦方向へ並べる。
- 1列表示では人物との左右関係がなくなるため、吹き出し先端は非表示にし、可読性と余白を優先する。
- 6件の表示順はDesktopと同じ1〜6の順序を維持する。

## 削除要素

セクション末尾の次のコピーと、そのラッパー要素を削除する。

- AIを「収入に直結するスキル」に変える、実践型スクール。
- 副業にも、キャリアにも。生成AIが、あなたの武器になります。

削除後に不要となる `.problemConclusion` のCSSも削除する。

## 実装境界

- 主な変更対象は `components/sections/ProblemsSection.tsx` と `app/page.module.css` とする。
- 既存の `SectionFrame`、`SafeImage`、`ASSETS.problemIllustration` を維持する。
- セクション2以外の文言、人物素材、CTA、他セクションの配置は変更しない。
- 吹き出しはHTML/CSSで実装し、画像化しない。これにより文字の可読性、アクセシビリティ、レスポンシブ性を維持する。

## テスト

- 新見出しがレベル2見出しとして1件表示されること。
- 旧見出しと旧導入文が存在しないこと。
- 新しい悩み6件が指定順で表示されること。
- 旧悩み4件が存在しないこと。
- Desktop用に左3件、人物、右3件の意味的なグループが存在すること。
- 吹き出しが人物方向を向く左右別クラスを持つこと。
- Mobile用CSSが1列順序、人物中央、吹き出し先端非表示を定義すること。
- 旧結論コピーと `.problemConclusion` が削除されていること。
- 全テスト、型検査、本番ビルドを実行すること。
- 実ブラウザでDesktopと390px幅を確認し、見出し・人物・吹き出しが重ならず横にはみ出さないことを確認すること。

## セキュリティと品質

- 外部通信、解析コード、認証情報を追加しない。
- 未使用import・未使用変数を残さない。
- 新しい実行時エラー経路は追加しないため、既存の `SafeImage` による画像エラーハンドリングを維持する。
