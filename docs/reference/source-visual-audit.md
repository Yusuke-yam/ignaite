# 参照LP ビジュアル監査

- 参照: `https://ai.digital-hacks.jp/`
- 監査日: 2026-06-19
- 対象: 実装側の13セクション（`hero` / `campaign` / `problems` / `about` / `reviews` / `feature` / `stories` / `curriculum` / `price` / `flow` / `faq` / `final-cta` / `footer`）
- 方法: 実ブラウザ上で DOM、computed style、要素矩形、表示画像を読み取り専用で測定

## 1. 実装で最初に固定する共通仕様

### カラートークン

| 用途 | 色 |
|---|---|
| ブランド赤 | `#ff282d` (`rgb(255, 40, 45)`) |
| 本文・見出し | `#222222` |
| 白 | `#ffffff` |
| 淡いピンク背景 | `#fff7f7` |
| 口コミカード内の濃赤 | `#850507` |
| 注釈 | `#5c5c5c` |
| 薄いピンク線・面 | `#fce1e4` 前後 |

アクセント色はほぼブランド赤1色。黄色はヒーロー見出しのマーカーでのみ強く使う。濃い黒を使わず、本文は `#222` を基本とする。

### 書体

- ヒーロー見出し: `Zen Kaku Gothic New`, sans-serif、700
- その他の和文: `Noto Sans JP`, sans-serif
- 欧文ラベル（Features / Flow / FAQ / Price など）も Noto Sans JP 系。文字間隔を広く取り、装飾線と合わせる。
- 見出しの基本 letter-spacing: `0.05em`。小さなヒーローバッジのみ `0.10em`。

### ブレークポイント

参照LP自身が次の3区分を持つ。

- Desktop: `min-width: 1200px`
- Tablet: `810px–1199.98px`
- Mobile: `max-width: 809.98px`

したがって 768px と 390px は同じ Mobile variant。768pxでも単なる縮小ではなく、縦積みへ切り替わる。

### 横幅・角丸・余白

- 1280px: 大カード外側余白 24px（幅1232px）、本文の標準コンテナ 1000px（x=140px）。
- 768px: 大カード外側余白 16px（幅736px）、テーブル等は24px（幅720px）。
- 390px: Mobile variant のため左右16pxを基準にし、カード幅358px。横スクロール対象だけ幅を保持する。
- 大カード角丸: 40px。
- 比較表角丸: 12px。
- Desktopの大カード内縦padding: 100px。Mobileは主要カードで16pxまで圧縮するが、内容ブロック間の縦間隔は広いまま。
- セクション見出しは中央揃えが基本。本文ブロックは最大1000px程度。

## 3. 13セクション別仕様

### 01 `hero`

正確な文言:

- バッジ: `副業` / `転職` / `キャリアアップ`
- H1: `未経験から最短で、` / `AI活用のプロになる。`
- リード: `生成AIをあなたの武器に変える実践型スクール。未経験からでも、着実にAIスキルを身につけて収入アップとキャリア成長を同時に実現します。`
- 実績: `総受講生数 3500名以上` / `副業成功率 92.8% ※1` / `受講生満足度 94.0% ※2`
- 注釈: `※1 2024年2月卒業生アンケートから集計` / `※2 2025年4月受講生アンケートから集計`

1280px:

- 高さ768px。左約56%、右約44%の2カラム。
- 左テキスト `#mv-txt`: x=0, y=161, w≈721, h≈445, padding `8px 8px 24px 80px`。
- H1: x=80, w=546, 45px / 49.5px、700、letter-spacing 2.25px。1行目赤、2行目`#222`。
- 2行目の下に黄色の幅広マーカー。テキストより少し長く、厚さ約10–12px。
- 右は人物写真を画面端までcover。左右境界は白い斜めカット。
- 左背景に非常に薄い巨大ロゴ形状と24px前後の方眼。
- ヘッダーは上中央の白いピル。ロゴ＋`生成AI`タグ、右端にハンバーガー。幅約300px、高さ52px、薄い影。
- 下中央にキャンペーンの固定ピルが一部重なる。実装では外部遷移を無効化しても形状は維持。

768px / 390px:

- 768pxでは高さ800px、`#mv-txt` x=16, y=100, w=736, h≈367, padding 8px、中央揃え。
- タイトルは約38px / 45.6px、700。1行目と2行目が別段。390pxでは同じMobile variantで32–38pxのclampを使い、左右16px内に収める。
- リードは12px / 18px、中央揃え。
- バッジは12px / 14.4px、900、白文字＋赤い台形ラベル。
- 写真は下側へ回し、人物の顔とノートPCが欠けないよう `object-position: 65% center` 前後。

素材:

- 人物切り抜き/写真: `hero-person.png`（参照元末尾 `lUelwMdwxxjejqM03BlEzcCZbHM.png`）
- 背景/共通グラフィック: `hero-background.png`（参照元末尾 `dwKqoNe40spQooP98mwkRbdDO0.png`）

### 05 `reviews`

- 見出し: `累計口コミ数 3万件以上`
- 赤〜淡いピンク背景上に、X投稿風のカードを大量に斜め/横方向へ流すマルキー。

- `挑戦を、カジュアルに。`
- ナビ: `デジハクとは` / `特徴` / `料金` / `流れ` / `FAQ`

- `Copyright © 2025 UNIT BASE, Inc. All rights reserved.`

見た目:

- 白または淡ピンク背景、赤いロゴ、濃色本文。
- Desktopはロゴ/タグライン、ナビ、CTAを横配置。下段に規約リンクとSNS。
- Mobileは中央揃えの縦積み。SNSは4アイコン（Instagram / X / YouTube / note）。
- 外部遷移は無効化対象。アンカーの視覚表現だけ維持する。

## 5. レスポンシブ差分の実装優先順位

1. 810px未満でHeroを中央揃えにし、写真を右カラム固定から下/背景へ切り替える。
2. Problemsの2列を1列にする。
3. Aboutを文章→画像の順で縦積み。
4. Featureの大カード幅を `calc(100% - 32px)` にし、内paddingを16pxへ。
5. Curriculumのタブは横スクロール可。講師カード帯はoverflow hiddenのまま。
6. Priceの比較表は縮小せず横スクロール。重要列（デジハク）は左側に固定すると見た目が近い。
7. Flowを3列から1列へ。ここがMobileのページ長増加の最大要因。
8. CTAの2ボタンを縦積み。FAQとFooterの左右レイアウトも縦積み。

## 6. 高類似度のための注意点

- セクション間を均等なpaddingで処理しない。参照はCTA前後と口コミ面に大きな「間」がある。
- 白カードは影より角丸と赤面とのコントラストが主体。影を強くしすぎない。
- 人物カード/生成物サムネイルは整然と並べず、斜め移動・重なり・画面外の切れを残す。
- Heroの斜め境界、黄色マーカー、薄い巨大ロゴ、方眼背景が第一印象を決める。
- 料金・キャンペーンの細かな文字入りビジュアルは既存画像を使う。再組版すると95%到達が難しい。
- FAQ、CTA、Footerの文言は現在の仮コンテンツと異なるため、上記の参照文言へ置き換える。
